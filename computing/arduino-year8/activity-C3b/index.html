Level 1 — Numbers from an analogue input

What are we doing in this level, sir?

You are going to learn how to use the serial monitor to see what an analogue input is doing.

Open a new Arduino window with a blank sketch in it.

Copy the sketch below into your Arduino window. Remember to delete the blank sketch first.

int sensorPin = A5;            // the input pin for the potentiometer on the ThinkerShield
int sensorValue = 0;           // variable to store the value coming from the sensor

void setup() {
    pinMode(12, OUTPUT);   // declare the ledPin as an OUTPUT
    Serial.begin(9600);        // set the speed of the serial data connection
}

void loop() {
    sensorValue = analogRead(sensorPin);     // read the value from the sensor
    Serial.print("The sensor value is: ");   // send the words over the serial data connection
    Serial.println(sensorValue);             // send the sensor value over the serial data connection
    delay(100);                              // wait for 0.1 seconds so that we only send the sensor value back 10 times a second.
}

Upload this sketch to your Arduino and check that it works.

Now open the Arduino “serial monitor”. You do this by choosing “Serial monitor” from the “Tools” menu, as shown in the picture.

Adjust the potentiometer (the knob) on the Arduino and look at the data coming in on the serial monitor.

What are those numbers, sir?

The number is how the Arduino knows the position of the potentiometer (or knob). This kind of input is called analogue.

Digital data is just “on” or “off”, HIGH or LOW, etc.

Analogue data is a number. There’s a minimum and a maximum possible value for that number. What are the smallest and biggest numbers you can get?

Level 2 — Controlling the LED with an analogue input

What are we doing in this level, sir?

You’re going to use the potentiometer to control how fast one of the LEDs flashes.

You will be changing the sketch from level 1, so keep it open and save it with a new name: C3b-level2.

If you can’t find it, you can copy it from here.

int sensorPin = A5             // the input pin for the potentiometer on the ThinkerShield
int sensorValue = 0;           // variable to store the value coming from the sensor

void setup() {
    pinMode(12, OUTPUT);   // declare the ledPin as an OUTPUT
    Serial.begin(9600);        // set the speed of the serial data connection
}

void loop() {
    sensorValue = analogRead(sensorPin);     // read the value from the sensor
    Serial.print("The sensor value is: ");   // send the words over the serial data connection
    Serial.println(sensorValue);             // send the sensor value over the serial data connection
    delay(100);                              // wait for 0.1 seconds so that we only send the sensor value back 10 times a second.
}

First, get rid of the line that says

    delay(100);                              // wait for 0.1 seconds so that we only send the sensor value back 10 times a second.

That was just there to slow down the program a bit, to make it go at a speed people can handle.

Do you remember changing the amount of time that the LED blinked on and off? You’ll copy some code from that.

Here is the complete sketch, but don’t copy anything yet.

void setup() {          // the setup function runs once when you press reset or power the board
pinMode(12, OUTPUT);     // initialize digital pin 12 as an output.
}
void loop() {           // the loop function runs over and over again forever
digitalWrite(12, HIGH);  // turn the LED on
delay(1000);             // wait for a second
digitalWrite(12, LOW);   // turn the LED off
delay(1000);             // wait for a second
}

Copy the code inside the setup() function. Put it at the end of the setup() function in your current C3b-level2 sketch.

Copy the code inside the loop() function. Put it at the end of the loop() function in your current C3b-level2 sketch.

Now the LED will blink on and off for one second, and the numbers from the potentiometer will come to the serial monitor.

Try it and see.

Now see if you can use the variable sensorValue to set the time in the delay() functions.

Here’s some pseudocode that might help you:

  1. Read the potentiometer
  2. Turn on the LED
  3. Wait for a delay time according to the value from the potentiometer
  4. Turn off the LED
  5. Wait for a delay time according to the value from the potentiometer

I can give you some hints if you need them, but have a go by yourself first.

The number in the brackets in the delay() function tells the Arduino how long to delay. But it doesn’t have to be a number – you can put a variable there too.

Change each delay(1000); to be delay(sensorValue);.

Level 3 — Turn LEDs on and off with the potentiometer

What are we doing in this level, sir?

You’re going to turn different LEDs on and off using the potentiometer.

Copy this sketch

int sensorPin = A5             // the input pin for the potentiometer on the ThinkerShield
int sensorValue = 0;           // variable to store the value coming from the sensor

void setup() {
    pinMode(8, OUTPUT);        // declare pin 8 as an OUTPUT
    pinMode(9, OUTPUT);        // declare pin 9 as an OUTPUT
    pinMode(10, OUTPUT);       // declare pin 10 as an OUTPUT
    pinMode(11, OUTPUT);       // declare pin 11 as an OUTPUT
    pinMode(12, OUTPUT);       // declare pin 12 as an OUTPUT
}

void loop() {
    sensorValue = analogRead(sensorPin);     // read the value from the sensor
}

Here is some pseudocode that tells you how the LEDs should be turned on and off. You’ll need to change this to Arduino code and put it in the loop() function.

  1. Turn off all the LEDs.
  2. If the value from the potentiometer is between 0 and 204, turn LED 8 on.
  3. If the value from the potentiometer is between 205 and 410, turn LED 9 on.
  4. If the value from the potentiometer is between 411 and 615, turn LED 10 on.
  5. If the value from the potentiometer is between 616 and 820, turn LED 11 on.
  6. If the value from the potentiometer is between 821 and 1023, turn LED 12 on.

Here is some code to turn LED 10 on if the value from the potentiometer is between 300 and 400.

    if (sensorValue >= 300 && sensorValue <= 400) {  // if the value from the potentiometer is between 300 and 400.
        digitalWrite(10, HIGH);                       // turn LED 10 on
    }

How would you turn all of the LEDs off? digitalWrite(8, LOW); turns off LED 8.

Pseudocode “If the value from the potentiometer is between 0 and 204, turn LED 8 on.” needs three lines of actual code:

    if (sensorValue >= 0 && sensorValue <= 204) {  // if the value from the potentiometer is between 0 and 204.
        digitalWrite(8, HIGH);                      // turn LED 8 on
    }
