Level 1 — Numbers from an analogue input

What are we doing in this level, sir?

You are going to learn how to use the serial monitor to see what an analogue input is doing.

Open a new Arduino window with a blank sketch in it.

Copy the sketch below into your Arduino window. Remember to delete the blank sketch first.

int sensorPin = A5             // the input pin for the potentiometer on the ThinkerShield
int sensorValue = 0;           // variable to store the value coming from the sensor

void setup() {
    pinMode(ledPin, OUTPUT);   // declare the ledPin as an OUTPUT
    Serial.begin(9600);        // set the speed of the serial data connection
}

void loop() {
    sensorValue = analogRead(sensorPin);     // read the value from the sensor
    Serial.print("The sensor value is: ");   // send the words over the serial data connection
    Serial.println(sensorValue);             // send the sensor value over the serial data connection
    delay(100);                              // wait for 0.1 seconds so that we only send the sensor value back 10 times a second.
}

Upload this sketch to your Arduino and check that it works.

Now open the Arduino “serial monitor”. You do this by choosing “Serial monitor” from the “Tools” menu, as shown in the picture.

Adjust the potentiometer (the knob) on the Arduino and look at the data coming in on the serial monitor.

What are those numbers, sir?

The number is how the Arduino knows the position of the potentiometer (or knob). This kind of input is called analogue.

Digital data is just “on” or “off”, HIGH or LOW, etc.

Analogue data is a number. There’s a minimum and a maximum possible value for that number. What are the smallest and biggest numbers you can get?

Level 2 —

What are we doing in this level, sir?

You’re going to use the potentiometer to control how fast one of the LEDs flashes.

You will be changing the sketch from level 1, so keep it open and save it with a new name: C3b-level2.

Do you remember changing the amount of time that the LED blinked on and off? Here is that sketch.

void setup() {          // the setup function runs once when you press reset or power the board
pinMode(12, OUTPUT);    // initialize digital pin 12 as an output.
}
void loop() {           // the loop function runs over and over again forever
digitalWrite(12, HIGH); // turn the LED on
delay(1000);            // wait for a second
digitalWrite(12, LOW);  // turn the LED off
delay(1000);            // wait for a second
}

You need to take parts of this sketch and add them to your C3b-level2 sketch.

First, see if you can get the LED to blink, and also see the numbers from the potentiometer in your serial monitor window.

I can give you some hints if you need them, but have a go by yourself first.

Your sketch already has everything to read the potentiometer input and send it to the serial monitor.

You also need to add all of the code I just showed you, but be careful: you can’t just put it all at the end.

Add the code inside the setup() function above to the setup() function in your sketch.

Then add the code inside the loop() function to the loop() function in your sketch.

Now that you are both reading the potentiometer analogue input and blinking the LED, you need to use the analogue input to control the time that the LED is on and off.

You have a variable named sensorValue that has the value from the potentiometer. Use this value to control how long the LED stays on, and also how long it stays off. Use sensorValue as the number of milliseconds.

I can give you some hints if you need them, but have a go by yourself first.

Do you remember that the two delay() function calls control how long the LED remains on and off? Use sensorValue to supply the number to delay().

So far you have put numbers into the delay() function call. Put the sensorValue variable in there instead.

Change each delay(1000) to be delay(sensorValue).
